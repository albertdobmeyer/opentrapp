**Date:** 2026-04-25 (revised same day after F11/F13/F14 closure + Tier 4 stress-test replay)
**Status:** Ready for tag, pending user action on `git tag v0.2.0 && git push origin v0.2.0`

## TL;DR

v0.2.0 ships **prosumer-targeted, Split-Shell-default, with curated use-case gallery and the F11/F13/F14 findings closed in the same release.** The product is provably defensible at all four layers (container + tool + LLM + operational). Soft Shell is verified-safe (Tier 4 stress test) but not yet user-flippable in the GUI — that surfaces in v0.3.0. Other advertised capabilities (vault-calendar, vault-voice) are still deferred to v0.3.0+.

### Three findings — ALL CLOSED in this release

| # | Sev | Finding | Status |
|---|---|---|---|
| F11 | HIGH | Soft Shell config doesn't deliver tools (root cause: `web_fetch` etc. have `profiles: []`, not in coding profile) | **CLOSED** — submodule `9e65672`: manifest gained `also_allow:` for soft preset; generator emits `tools.alsoAllow`. Live verification: bot at Soft Shell now reports access to web_search, web_fetch, canvas, message, process. |
| F14 | MEDIUM | `tool-control.sh --apply` creates a rogue parallel container | **CLOSED** — submodule `02e6718`: `--no-restart` flag gates compose lifecycle; parent orchestrator owns restart. |
| F13 | LOW | `tool-control.sh` hardcodes pre-v2-perimeter container name | **CLOSED** — same submodule commit as F14: `OPENCLAW_CONTAINER` env override. |
| F12 | MEDIUM | Soft Shell defensive verbosity degraded | **FALSIFIED + downgraded** — 8 controlled trials prove warning behavior is prompt-phrasing-dependent, not shell-dependent. See `tests/e2e-telegram/F12-VERDICT-2026-04-25.md`. |

For v0.2.0 ship: Split Shell remains the GUI default. Soft Shell is fully functional and verified-safe (Tier 4: 19/19 attacks blocked) but the GUI shell-switcher surface is deferred to v0.3.0 to keep this release focused on what was originally promised.
